## プロジェクト概要
本プロジェクトは、Kaggleの実データ「E-commerce A/B Testing Dataset」を使用し、Webサイトのデザイン変更が購入率に与える影響を統計的に検証したものです。

## 目的
- 実データ特有のノイズ（設定ミスや重複ログ）のクレンジング
- カイ二乗検定を用いたA/Bテストの有意差判定
- ロジスティック回帰分析（GLM）による要因分析とオッズの解釈
- 統計的根拠に基づいたビジネス上の意思決定と提案

## 使用技術
- **言語:** Python 3
- **主要ライブラリ:** pandas, numpy, scipy (stats), statsmodels
- **環境:** Google Colaboratory

## 分析アプローチと主な発見

### 1. データの前処理（クレンジング）
生データ（約30万件）には、システムエラーによる「Control群にNew Pageが表示される」などの矛盾したログや、同一ユーザーの重複アクセスが含まれていました。これらを除外・整理し、約29万件のクリーンな分析用データを作成しました。

### 2. 仮説検定（カイ二乗検定）による評価
グループごとのコンバージョン率を集計すると、以下のようになりました。
- **旧ページ (Control):** 約 12.04%
- **新ページ (Treatment):** 約 11.88%

一見すると旧ページの方が高く見えますが、これが統計的に意味のある差（有意差）なのかを確認するためカイ二乗検定を実施しました。結果、**p値は 0.05 を上回り**、コンバージョン率の差は「誤差の範囲内」であることが確認されました。

### 3. 一般化線形モデル (GLM) - ロジスティック回帰分析
コンバージョン（する/しない）という二項分布に従う結果を予測するため、`statsmodels` を用いてロジスティック回帰分析を行いました。

- **分析結果:** `ab_page`（新ページを見たかどうか）の係数は **-0.0150**、p値は **0.1899** となりました。
- **解釈:** p値が有意水準（0.05）を大きく上回っているため、新しいページがコンバージョンに与える影響は統計的に有意ではありません。また、係数がマイナスであることから、確率を上げる効果は全くないことがモデルからも裏付けられました。

## ビジネスへの提言
本分析の結果から、**「新しいWebページのデザイン案は、本番環境に導入すべきではない（現行維持が妥当）」**と結論付けます。
有意差のない新ページをリリースするためのエンジニアリングリソース（時間やコスト）を削減し、価格設定や別の導線改善など、別のアプローチでの施策検討にリソースを投資することを推奨します。
